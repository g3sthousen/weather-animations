# Security Policy

## Reporting a Vulnerability

Please do not report security issues as public GitHub issues.

If GitHub private vulnerability reporting is enabled for this repository, use that flow. If it is not enabled yet, open a minimal public issue that asks for a private contact channel without disclosing vulnerability details.

When sharing details privately, include:

- A short description of the vulnerability.
- Steps to reproduce or verify the issue.
- The affected version or commit, if known.
- Any suggested mitigation.

The maintainer will review the report and coordinate a fix before public disclosure when appropriate.

## Supported Versions

Until the project reaches a stable `1.0.0` release, security fixes target the latest published version only.
